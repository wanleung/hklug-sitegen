Title: OpenBSD 7.9 Raises CPU Core Limit to 255, Adds WiFi 6 Support
Content:
<p>The OpenBSD project has announced the release of OpenBSD 7.9, marking another significant milestone for the security-focused BSD operating system. Version 7.9 introduces substantial hardware support improvements, including expanded CPU core scalability and initial WiFi 6 compatibility.</p>
<h4>Maximum CPU Core Support Expanded to 255<//h4>
<p>One of the headline features in OpenBSD 7.9 is the substantial increase in maximum CPU core support for AMD64 (x86_64) systems. The operating system now supports up to 255 CPU cores, a significant jump from the previous limit of 64 cores. This change brings OpenBSD closer to parity with other modern operating systems as high-core-count processors become increasingly common in both workstation and server environments.</p>
<p>According to project lead Theo de Raadt, the 255-core ceiling stems from xAPIC architectural requirements. While contemporary dual-socket Intel Xeon and AMD EPYC processors can exceed this core count, OpenBSD must work within these constraints until x2APIC support reaches production readiness. The development team continues to work on improved x2APIC implementation for future releases.</p>
<h4>Initial WiFi 6 Support<//h4>
<p>OpenBSD 7.9 introduces initial support for 802.11ax wireless networking, commonly known as WiFi 6. This addition represents a meaningful step forward for the platform's wireless capabilities, enabling compatibility with modern wireless networking hardware. The implementation integrates with OpenBSD's existing wireless stack, maintaining the project's commitment to security and code quality even as it expands hardware support.</p>
<p>The WiFi 6 support in this release provides users with access to improved wireless throughput and efficiency compared to previous generations, though the initial implementation focuses on establishing a stable foundation rather than exhaustive hardware compatibility.</p>
<h4>Additional Hardware and Driver Improvements<//h4>
<p>Beyond the headline features, OpenBSD 7.9 includes numerous hardware driver enhancements and system improvements. The AMDGPU graphics driver receives important fixes, improving stability and compatibility for systems with AMD graphics hardware. The release also addresses floating-point state leakage vulnerabilities affecting AMD Zen 1 processors, reinforcing OpenBSD's security-first approach.</p>
<p>Network connectivity receives attention with the Intel ICE Ethernet driver now enabled on ARM64 platforms, expanding hardware options for users deploying OpenBSD on ARM-based systems. The release also incorporates scheduler improvements that enhance overall system responsiveness and performance under varied workload conditions.</p>
<h4>Delayed Hibernation Support<//h4>
<p>Version 7.9 introduces support for delayed hibernation, providing users with greater flexibility in power management. This feature allows systems to schedule hibernation events, enabling more sophisticated power management strategies for both desktop and server deployments.</p>
<h4>Availability and Documentation<//h4>
<p>OpenBSD 7.9 is available now for download from the project's official website. The release includes comprehensive documentation detailing the full range of changes, security improvements, and hardware compatibility updates. As with all OpenBSD releases, version 7.9 emphasizes code correctness, security, and stability over feature proliferation, maintaining the project's distinctive philosophy in the BSD ecosystem.</p>
<p>The release continues OpenBSD's tradition of regular six-month release cycles, providing users with predictable update schedules and consistent access to security improvements and hardware support enhancements.</p>

<h4>最大 CPU 核心支援擴展至 255<//h4>
<p>OpenBSD 7.9 其中一個重點功能，就係大幅擴大 AMD64 (x86_64) 系統嘅最大 CPU 核心支援。而家個作業系統最多可以支援 255 個 CPU 核心，比起之前嘅 64 個核心上限有顯著提升。隨住高核心數處理器喺工作站同伺服器環境越來越普及，呢個改動令 OpenBSD 更接近其他現代作業系統嘅水平。</p>
<p>根據項目負責人 Theo de Raadt 嘅解釋，255 個核心嘅上限係源於 xAPIC 嘅架構要求。雖然而家嘅雙插槽 Intel Xeon 同 AMD EPYC 處理器可以超過呢個核心數，但喺 x2APIC 支援達到生產環境可用之前，OpenBSD 必須喺呢個限制下運作。開發團隊正繼續努力改善 x2APIC 嘅實作，預計會喺未來版本推出。</p>
<h4>初步支援 WiFi 6<//h4>
<p>OpenBSD 7.9 首度引入對 802.11ax 無線網絡嘅支援，也就是大家熟悉嘅 WiFi 6。呢個升級對平台嘅無線能力嚟講係重要一步，令到可以相容現代無線網絡硬件。實作方面會整合入 OpenBSD 現有嘅無線網絡堆疊，即使擴展硬件支援，都維持項目對安全同程式碼質素嘅堅持。</p>
<p>呢個版本嘅 WiFi 6 支援令用戶可以享受到比起前代更好嘅無線吞吐量同效率，不過初步實作主要係建立穩定基礎，而唔係追求全面嘅硬件相容性。</p>
<h4>其他硬件同驅動程式改進<//h4>
<p>除咗上述重點功能，OpenBSD 7.9 仲包含大量硬件驅動程式增強同系統改進。AMDGPU 顯示卡驅動程式收到重要修復，改善咗使用 AMD 顯示卡系統嘅穩定性同相容性。版本亦處理咗影響 AMD Zen 1 處理器嘅浮點狀態洩漏漏洞，進一步鞏固 OpenBSD 安全優先嘅理念。</p>
<p>網絡連接方面，Intel ICE 以太網驅動程式而家已經喺 ARM64 平台啟用，擴展咗喺 ARM 系統部署 OpenBSD 嘅硬件選擇。版本仲加入排程器改進，喺不同工作負載情況下提升整體系統嘅回應速度同效能。</p>
<h4>延遲休眠支援<//h4>
<p>7.9 版本引入延遲休眠支援，令用戶喺電源管理上有更大彈性。呢個功能允許系統排程休眠事件，令桌面同伺服器部署可以採用更進階嘅電源管理策略。</p>

<h4>最大 CPU 核心支援擴展至 255<//h4>
<p>OpenBSD 7.9 的亮點功能之一，是針對 AMD64（x86_64）系統的最大 CPU 核心支援數大幅增長。該作業系統現已支援高達 255 個 CPU 核心，較先前 64 核心的限制有顯著提升。隨著高核心數處理器在 workstation 與伺服器環境中日益普及，此項改動使 OpenBSD 更接近其他現代作業系統的水平。</p>
<p>根據專案負責人 Theo de Raadt 所述，255 核心的上限源於 xAPIC 的架構需求。儘管當代雙插槽的 Intel Xeon 與 AMD EPYC 處理器核心數可超過此限制，但在 x2APIC 支援達到生產環境可用狀態前，OpenBSD 仍須在此架構限制下運作。開發團隊正持續致力於改進 x2APIC 的實作，以期於未來版本中推出。</p>
<h4>初步 WiFi 6 支援<//h4>
<p>OpenBSD 7.9 首度引入對 802.11ax 無線網路的初步支援，即一般所稱的 WiFi 6。此項新增功能為該平台的無線能力帶來實質進展，使其能夠相容於現代無線網路硬體。該實作已整合至 OpenBSD 現有的無線網路堆疊中，即便持續擴展硬體支援，仍堅守專案對安全性與程式碼品質的承諾。</p>
<p>本次發布的 WiFi 6 支援為使用者帶來相較前代更優異的無線頻寬與效率；不過，初期實作主要著重於建立穩定的基礎，而非追求全面的硬體相容性。</p>
<h4>其他硬體與驅動程式改進<//h4>
<p>除上述重點功能外，OpenBSD 7.9 亦包含多項硬體驅動程式與系統改進。AMDGPU 顯示卡驅動程式已獲得重要修復，提升了搭載 AMD 顯示卡系統的穩定性與相容性。本次發布亦修復了影響 AMD Zen 1 處理器的浮點狀態洩漏漏洞，進一步強化 OpenBSD 以安全為優先的理念。</p>
<p>網路連線方面，Intel ICE 乙太網路驅動程式現已於 ARM64 平台啟用，為部署於 ARM 架構系統的使用者擴充了硬體選擇。本次發布亦納入排程器改進，於不同工作負載條件下提升整體系統的回應速度與效能。</p>
<h4>延遲休眠支援<//h4>
<p>7.9 版本引入了延遲休眠支援，為使用者在電源管理上提供更大彈性。此功能允許系統排程休眠事件，使桌面與伺服器部署能採用更進階的電源管理策略。</p>
