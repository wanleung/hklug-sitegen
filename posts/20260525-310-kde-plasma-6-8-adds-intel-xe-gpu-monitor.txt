Title: KDE Plasma 6.8 Adds Intel Xe GPU Monitoring to System Monitor, 6.7 Piles On Crash Fixes
Content:
<p>KDE developers are pushing forward on two fronts: Plasma 6.7 is targeting a mid-June release with a batch of crash fixes, while early feature work on Plasma 6.8 will bring Intel Xe GPU support to the desktop's System Monitor.</p>
<p>Plasma 6.7 is addressing several stability issues. The release includes fixes for crashes triggered by plugging or unplugging displays, when installing Flatpak apps from <code>.flatpakref</code> files, and when the Weather Report widget polls for updates. KWin has also received memory leak fixes as part of this release cycle.</p>
<p>The more notable development is reserved for Plasma 6.8: native Intel Xe kernel driver support within the System Monitor app and its desktop widgets. The addition will surface GPU telemetry for modern Intel integrated and discrete graphics directly in the Plasma interface. It has taken until now for this integration to arrive — a bug report requesting Xe support was only opened last year.</p>
<p>Users running AMD or NVIDIA hardware will continue to rely on Plasma's existing monitoring options, as the new widget targets the Xe driver specifically.</p>
<p>Beyond GPU monitoring, Plasma 6.8 will also change how Discover handles leftover Flatpak data: deleting settings for removed apps will now move them to the trash rather than permanently deleting them immediately.</p>
<p>Plasma 6.7 arrives next month as the next iteration in KDE's active development cycle. The Intel Xe monitoring feature in 6.8 fills a gap that Linux desktop users with Intel graphics have waited on for some time.

<p>KDE 開發人員正雙線推進：Plasma 6.7 目標六月中旬發佈，附帶一批當機修復；而 Plasma 6.8 的早期功能開發則會為桌面的系統監控器帶來 Intel Xe GPU 支援。</p>
<p>Plasma 6.7 正在處理多項穩定性問題。該版本包含修復插拔顯示器時、從 <code>.flatpakref</code> 檔案安裝 Flatpak 應用程式時，以及天氣報告 widget 輪詢更新時觸發的當機。KWin 亦於此發佈週期中獲得記憶體洩漏修復。</p>
<p>更引人注目的發展留給 Plasma 6.8：系統監控器應用程式及其桌面 widget 將加入原生 Intel Xe kernel driver 支援。此新增功能會直接在 Plasma 介面中顯示現代 Intel 集成和獨立顯示卡的 GPU telemetry。這項整合遲至現在才實現——一項要求 Xe 支援的 bug report 去年才開啟。</p>
<p>使用 AMD 或 NVIDIA 硬件的用戶將繼續依賴 Plasma 現有的監控選項，因為新 widget 專門針對 Xe driver。</p>
<p>除 GPU 監控外，Plasma 6.8 還會改變 Discover 處理殘留 Flatpak 數據的方式：刪除已移除應用程式的設定時，現在會移至垃圾桶，而非立即永久刪除。</p>
<p>Plasma 6.7 將於下月推出，作為 KDE 活躍開發週期的下一階段。6.8 中的 Intel Xe 監控功能填補了使用 Intel 顯示卡的 Linux 桌面用戶長期以來的需求。</p>
