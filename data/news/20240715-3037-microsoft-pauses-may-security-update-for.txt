Title: Microsoft Pauses May Security Update for Some Dell Devices Citing Shutdown Risks
Content:
<p>Microsoft is preventing its latest Windows 11 security update from installing on certain Dell computers after discovering it can trigger unexpected shutdowns and severe performance issues. The company has deployed a "safeguard hold" to automatically block the problematic patch from reaching the at-risk machines.</p>
<p>The problematic update is the May 2024 security cumulative update, designated KB5037765. As detailed in a BleepingComputer report, a specific subset of Dell devices experiences system instability and performance degradation after installing it. Consequently, Microsoft has proactively intervened, placing a technical block that prevents Windows Update from offering the release to these specific hardware configurations.</p>
<p>This automatic safeguard hold is designed as a safety net. Users with affected Dell devices will not see the update available for installation. They can verify the status of their system and any holds via the official Microsoft Windows Release Health dashboard, which tracks known update issues.</p>
<p>For users on the held-back devices, the directive is to take no immediate action. The safeguard hold ensures the update cannot install automatically. Microsoft is developing a corrected version of the patch, which will be released once compatibility is confirmed. The hold will then be lifted to allow the secure update to proceed.</p>
<p>The situation underscores the perennial trade-off between immediate system stability and security patching. By pausing the update, Microsoft averts the more disruptive outcome of device failure. However, this leaves the affected Dell systems temporarily unpatched against the security flaws addressed in the May release, a period of exposure that will last until the fixed update is deployed.</p>
<p>This incident is part of a broader pattern of update complexities in the diverse Windows ecosystem. The report from BleepingComputer marks it as the second significant issue identified during this update cycle, fueling recurring concerns about the rigor of pre-release testing for large-scale patches.</p>
<p>As of the latest information, Microsoft has not specified which Dell models are impacted or provided a timeline for delivering a resolved update. In the interim, affected users and administrators must rely on the Windows Release Health dashboard for status updates and wait for an official all-clear before proceeding with their security updates.</p>

<p>微軟暫停向部分戴爾電腦推送最新的Windows 11安全更新，因發現此更新可能導致意外關機及嚴重性能問題。該公司已部署「安全防護鎖定」機制，自動阻止有風險的電腦安裝此問題修補程式。</p>
<p>此問題更新為2024年5月安全累積更新，編號KB5037765。據BleepingComputer報導指出，特定型號的戴爾裝置在安裝後出現系統不穩定及性能下降問題。因此微軟主動介入，透過技術封鎖措施，阻止Windows Update向這些特定硬件配置提供該更新。</p>
<p>此自動安全防護鎖定被設計為安全網。受影響的戴爾裝置用戶將不會看到可安裝的更新。他們可透過官方微軟Windows版本健康狀況儀表板驗證系統狀態及任何鎖定情況，該儀表板會追蹤已知的更新問題。</p>
<p>對於被鎖定裝置的用戶，指示是無需立即採取行動。安全防護鎖定確保更新無法自動安裝。微軟正在開發修正版的修補程式，待兼容性確認後將予發布，屆時將解除鎖定以允許安全更新進行。</p>
<p>此情況凸顯了即時系統穩定性與安全修補之間的永恆權衡。透過暫停更新，微軟避免了更嚴重的設備故障後果。然而，這使得受影響的戴爾系統在安全缺陷修補方面暫時未受保護，此暴露期將持續到修正更新部署為止。</p>
<p>此事件反映了多元Windows生態系統中更廣泛的更新複雜性模式。BleepingComputer的報告標記這是此更新週期中發現的第二個重大問題，引發了對大規模修補程式發佈前測試嚴謹性的反覆關注。</p>
<p>根據最新資訊，微軟尚未指定受影響的戴爾型號或提供交付已解決更新的時間表。在此期間，受影響的用戶和管理員必須依賴Windows版本健康狀況儀表板獲取狀態更新，並等待官方確認安全後才繼續進行安全更新。</p>
