Title: Linux Kernel 7.2-rc1 Released, Marking Start of Final Stabilization Cycle
Content:
<p>The first release candidate for Linux kernel 7.2 is now available, officially closing the feature merge window and launching the final eight-week stabilization phase before the production release. This version is poised to become the core of upcoming major distributions, including Fedora 45 and Ubuntu 26.10.</p>
<p>Linus Torvalds announced the 7.2-rc1 pre-release, stating that overall, "things look reasonably normal." This signals a smooth transition from the active development period into a focused cycle of bug fixing and refinement, a critical process for ensuring kernel reliability.</p>
<p>A primary technical highlight is enhanced AMD hardware enablement. The update introduces support for HDMI 2.1 FRL (Fixed Rate Link) within the AMDGPU driver. FRL is a high-speed signaling technology necessary for driving features such as high-refresh-rate monitors and 8K displays, and its inclusion ensures the open-source AMD driver stack remains compatible with modern display standards.</p>
<p>The merge window also incorporated support for AMD's ISP4 and CAS platform components. These additions advance the integration of the company's latest system architecture into the mainline kernel, contributing to better system stability and performance on AMD-based hardware.</p>
<p>With the release of 7.2-rc1, Linux enters its standard eight-week testing window. Developers and early adopters will now rigorously test the kernel, with subsequent release candidates addressing any issues that arise. This meticulous stabilization cycle is essential for delivering a dependable final product to the wider user base.</p>
<p>This milestone exemplifies the methodical progress that sustains the Linux ecosystem. While an incremental step, the careful integration of modern hardware support and the strict adherence to release discipline provide the foundation for robust, capable Linux distributions in the coming months.</p>
<hr>
<p>Linux 內核 7.2 的首個發佈候選版本現已推出，正式關閉功能合併窗口，並啟動正式版發佈前為期八週的最後穩定化階段。此版本有望成為即將推出的主要發行版的核心，包括 Fedora 45 及 Ubuntu 26.10。</p>
<p>Linus Torvalds 公佈了 7.2-rc1 預發佈版本，並表示整體而言「情況看起來相當正常」。這標誌著從活躍開發期平穩過渡到專注於除錯與優化的週期，這是確保內核可靠性的關鍵過程。</p>
<p>一個主要的技術亮點是加強了對 AMD 硬件的支援。此次更新在 AMDGPU 驅動程式中引入了對 HDMI 2.1 FRL（固定速率鏈路）的支援。FRL 是一種高速信號傳輸技術，對於驅動高刷新率顯示器及 8K 顯示器等功能必不可少，其加入確保了開源 AMD 驅動程式堆疊與現代顯示標準的相容性。</p>
<p>合併窗口亦納入了對 AMD 的 ISP4 和 CAS 平台組件的支援。這些新增功能促進了該公司最新系統架構整合至主線內核，有助於提升基於 AMD 硬件的系統穩定性及性能。</p>
<p>隨著 7.2-rc1 的發佈，Linux 進入了標準的八週測試窗口。開發者及早期採用者將嚴格測試此內核，隨後的發佈候選版本將解決出現的任何問題。這種精細的穩定化週期對於向廣大用戶群提供可靠的最終產品至關重要。</p>
<p>此里程碑體現了維繫 Linux 生態系統的有條不紊進展。儘管這只是循序漸進的一步，但對現代硬件支援的謹慎整合及嚴格遵循發佈紀律，為未來數月內功能強健、能力出色的 Linux 發行版奠定了基礎。</p>
